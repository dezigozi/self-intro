# self-intro ポータル 運用・追加手順書

このドキュメントを読めば、Cursor のチャット履歴がなくても「新しい自己紹介HTMLを増やす」「ポータルに並べる」「デプロイする」ができるように書いてあります。

---

## 1. フォルダ構成と役割

```
self-intro/
├── index.html           ← ポータル（トップ）。ここにカードを増やす
├── self-intro_base.html ← バージョン1（図解ツールベース）
├── self-intro_gemini1.html
├── self-intro_gemini2.html
├── README.md            ← 公開URL・一覧・コマンドのメモ
└── 手順書.md            ← このファイル（追加・デプロイの手順）
```

- **index.html** … トップページ。アイコン（カード）をクリックすると各HTMLに飛ぶ。
- **self-intro_〇〇.html** … 実際の自己紹介ページ。増やすたびにポータルにカードを1枚追加する。

---

## 2. 新しい自己紹介HTMLを1つ追加する手順

### 2-1. 新しいHTMLファイルを用意する

- ファイル名の例: `self-intro_新バージョン名.html`（例: `self-intro_gemini3.html`）
- 中身は既存の `self-intro_base.html` などをコピーして編集してOK。

### 2-2. 新HTMLのフッターに「ポータルに戻る」「公開URL」を入れる

新しく追加したHTMLの `<footer>...</footer>` の中に、次の2つを入れる。

- **ポータルに戻る** … `href="index.html"`
- **公開URL** … `href="https://diagram-self-intro.surge.sh"`（target="_blank" 推奨）

既存の3つのHTMLのフッターをそのままコピーして、リンク部分だけ書き換えればよい。

例（テキストのみ）:

```html
<p class="text-xs mt-3">
  <a href="index.html" class="...">ポータルに戻る</a>　|　
  <a href="https://diagram-self-intro.surge.sh" class="..." target="_blank" rel="noopener">公開URL</a>
</p>
```

### 2-3. ポータル（index.html）にカードを1枚追加する

`self-intro/index.html` を開く。

- `<nav class="grid ...">` のなかにある、既存の `<a href="self-intro_〇〇.html">...</a>` のブロックをコピーする。
- そのブロックを、他のカードの下（または横）に貼り付ける。
- 次の3つを書き換える:
  1. **href** … 新しいファイル名（例: `self-intro_gemini3.html`）
  2. **アイコン** … `data-lucide="〇〇"` を変えたいアイコン名に（例: `sparkles`, `user`, `file-text` など。Lucide Icons の名前）
  3. **表示テキスト** … 「〇〇版」「Professional 〇〇」など、カードに表示する名前

例（追加する1ブロック）:

```html
<a href="self-intro_gemini3.html" class="group flex flex-col items-center p-6 md:p-8 bg-white rounded-2xl border border-slate-200 shadow-sm hover:shadow-md hover:border-violet-200 transition-all duration-200 text-center">
  <span class="flex items-center justify-center w-14 h-14 rounded-xl bg-violet-500/10 text-violet-600 mb-4 group-hover:bg-violet-500/20 transition-colors">
    <i data-lucide="book-open" class="w-7 h-7"></i>
  </span>
  <span class="font-bold text-slate-900">新バージョン名</span>
  <span class="text-xs text-slate-500 mt-1">短い説明</span>
</a>
```

- カードが4枚以上になる場合は、`sm:grid-cols-3` を `sm:grid-cols-4` などに変えるとレイアウトを調整できる。

### 2-4. README.md の一覧を更新する

`self-intro/README.md` の「フォルダ内のHTML」の表に、新しい1行を追加する。

- ファイル名
- 説明（何版か）
- 直リンク（`/ファイル名`）

## 3. デプロイ手順（Surge に公開する）

**self-intro フォルダごと** デプロイすると、トップがポータルになり、すべてのHTMLが同じドメインで見られる。

1. ターミナルを開く。
2. プロジェクトのルート（`creating-visual-explainers` があるフォルダ）に移動する。

```bash
cd "/Users/dezigozigmail.com/Dropbox/会社/仕事/マイドキュメント/Ｍ／G/生産性向上(働き方改革)/AI/■PB/AI School/creating-visual-explainers"
```

3. 次のコマンドを実行する。

```bash
npx --yes surge self-intro diagram-self-intro.surge.sh
```

4. 初回だけ Surge のログイン（メール・パスワード）を聞かれたら入力する。  
   2回目以降はそのままアップロードされる。

5. 成功すると次のURLで公開される。
   - ポータル: **https://diagram-self-intro.surge.sh/**
   - 各ページ: **https://diagram-self-intro.surge.sh/self-intro_〇〇.html**

---

## 4. サイトを削除したいとき

次のコマンドで、このドメインごと削除できる。

```bash
npx surge teardown diagram-self-intro.surge.sh
```

## 5. よく使う Lucide アイコン名（ポータル用）

ポータルのカードで使えるアイコン例（`data-lucide="ここ"` に指定）:

| 名前 | 用途の例 |
|------|----------|
| `layout` | 図解・レイアウト系 |
| `user` / `user-circle` | プロフィール |
| `sparkles` | おしゃれ・ポートフォリオ |
| `file-text` | ドキュメント風 |
| `book-open` | ストーリー・読み物 |
| `briefcase` | 仕事・キャリア |
| `palette` | デザイン系 |

一覧: https://lucide.dev/icons/
